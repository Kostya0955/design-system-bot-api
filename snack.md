
# Snack

## Что это
Краткое уведомление, которое появляется в ответ на действие пользователя и используется для мгновенной обратной связи. Появляется поверх всего интерфейса.

## Размеры и отступы
Рекомендумая максимальная высота для корректного отображения на экране — 76px (вмещает 2 строки текста при максимальной ширине). Однако при необходимости, можно увеличить высоту. 

Максимальная ширина — 720px. Не рекомендуется увеличивать максимальную ширину, так как это усложнит считываемость контента. 

Если текст не помещается в одну строку (например, его очень много или кнопка действия длинная), используйте несколько строк. Для этого установите Fill Container внутренним слоям компонента

## Выравнивание
Вертикальное выравнивание - по верхнему краю. Т.е. при наличии нескольких строк текста высота компонента увеличится, а положение иконки, кнопки действия и закрытия останутся неизменными

## Использование. Успешное выполнение (зеленая иконка success)
Сообщает пользователю, что его действие выполнено успешно.
Используйте кнопки, если нужно:
— дать доступ к результату; 
— отменить действие;  
— дать возможность повторить/сделать ещещеещеще;  
— напомнить про следующее логическое действие

## Использование. Предупреждение (желтая иконка warning)
Помогает предотвратить неправильные действия или обратить внимание на важные ограничения. Используйте кнопки с действием, если нужно: 
— принять меры, чтобы избежать негативных последствий;
— перейти к дополнительной информации или настройке;  
— отменить действие, если уже что-то пошло не так

## Использование. Ошибка (красный иконка error)
Информирует о том, что действие не удалось. Акцент делается на том, что произошло, и, по возможности, дается понимание, как исправить ситуацию.
Используйте кнопки, когда нужно:
— открыть страницу с подробностями;  
— изменить данные / настройки;  
— повторить действие;  
— связаться с поддержкой.

## Использование. Информирование (белая иконка info)
Используется для отображения нейтральных системных уведомлений. Не требует действий от пользователя, просто информирует о текущем статусе.
Используйте кнопки, когда нужно:
— открыть дополнительную информацию, перейти на другую страницу;  
— запустить/повторить действие

## Продолжительность действия
Используется для отображения длительного процесса. После окончания процесса snack loader исчезает, а на его месте появляется snack success/error.
Используйте кнопки, когда ожидание можно прервать

## Кнопки действия
По умолчанию кнопки в компоненте используются для перехода или для решения проблемы пользователя. Даже, если говорится про удаление какой-либо сущности — это не негативное для пользователя действие, это решение его проблемы. Поэтому по умолчанию они синего (accent) цвета. Однако, кнопки могут нести и негативный для пользователя характер. Например, прерывание процесса, который запустил сам пользователь. В таком случае используйте красные кнопки. Для этого перейдите в компонент Function button и установите свойство type = error.

## Появление и скроллинг
Компонент появляется внизу экрана и сохраняет свое положение при скроллинге

## Скрытие
Исчезает автоматически или вручную по кнопке закрытия

## Автоматическое скрытие
Используйте автоматическое скрытие, если snack:
— не содержит критически важной информации. В таком случае пользователю нет потребности тянутся к кнопке закрытия;
— не требует совершения важных действий от пользователя; 
— не перекрывает контент и не требует особого внимания.
По умолчанию snack исчезает через 3 секунды автоматически. Можно указать другое время исчезновения. Рекомендуемая максимальная продолжительность — 7 секунд.

Используйте минимальное время 3 секунды, для показа коротких сообщений в одну строку, которые не требуют особого внимания и действий от пользователя.

Используйте задержку 5 секунд для коротких сообщений с кнопкой действия или длинным текстом в несколько строк
Используйте задержку 7 секунд для длинных сообщений в несколько строк и кнопкой действия

Костя Головнёв, [09.12.2025 16:19]
## Остановка таймера
Если навести курсор на snack, то таймер остановит свою работу и компонент не пропадет даже если стоит ограничение в N секунды. Если убрать курсор, то таймер продолжит работу.

## Скрытие по кнопке
Если сообщение содержит важную информацию и требует от пользователя каких-то действий, лучше использовать кнопку закрытия. Так пользователь сам решит, когда ему станет не нужна эта информация

## Скролл
При скролле страницы snack сохраняет свое положение

## Количество снеков на экране
Рекомендуется одновреммено показывать 1 snack на экране и отображать новый только при исчезновении старого. Однако при необходимости может быть показано несколько снеков одновременно. В таком случае лучше использовать не более 2 снеков.

Новый снек появляется выше предыдущего

## Несколько строк текста 
По умолчанию компонент подстраивается под ширину контента. Однако, если текст не помещается в одну строку (например, его очень много, или кнопка действия очень длинная), используйте несколько строк. Для этого установите свойство Fill Container внутренним слоям компонента

## Как нельзя использовать
- Не рекомендуется использовать длинные тексты. Идеально, если они будут помещаться в одну строку. Допустимо 2 строки
- Не используйте snack для критических ошибок, мешающих работе пользователя. Для этого лучше использовать Modal window. Оно блокирует интерфейс пользователя и позволяет акцентировать внимание пользователя на проблеме
- Не используйте snack для уведомлений, которые не связаны напрямую с действиями пользователей и могут появиться фоном 
(например: новые сообщения, приглашения; уведомления о статуме фонового процесса). Для этого подойдет Toast Notification.
- Используйте компонент только внизу по центру экрана. Не располагайте его снизу слева или справа. Только по центру
- Не рекомендуется использовать больше 2 снеков одновременно. Пользователю будет сложно считывать информацию.
