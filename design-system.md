\# Loader

\## Когда использовать

Загрузка длится >400 мс и <20 сек;

Структура данных неизвестна или сервер не умеет отдавать прогресс;

Расположение в компактном компоненте (кнопка, строка таблицы, небольшая панель) или в целом блоке данных/странице


\## Отступы и ограничения

Отступ текст ↔️ loader = 8px (6px для размеров 2XS);

Текст max ширина = 250px;

Вертикально max 3 строки, горизонтально max 1 строка;

\## Размеры

•2XS (16px) — маленькие компоненты, деревья, иконки;

` `•XS (20px) — небольшие карточки, футеры, кнопки M;

` `•S (24px, базовый) — подгрузка карточек, блоков;

` `•M (36px) — большие блоки, страницы, боковые панели

\## Цвета

Синий — рядом с синим текстом;

⚫️ Чёрный — основной вариант. С черным текстом рядом;

🔴 Красный — только для деструктивных действий (удаление, отключение), рядом с красным текстом;

⚪️ Белый — рядом с белым текстом;

Другие цвета запрещены.

\## Loader без текста

Процесс до 2 сек;

Контейнер маленький (<50px);

Действие очевидно

\## Loader с текстом

3–5 сек → короткий текст («Загрузка данных…»);

5–20 сек → поясняющий текст («Подключаем сервис… может занять до 30 сек»);

Текст вертикально → если контейнер >120px высотой и >250px шириной, до 3 строк;

Текст горизонтально → если контейнер <120px или узкий, 1 строка

\## Поверх контента

действие критичное (отправка формы, удаление, оплата);

пользователь не должен менять данные во время процесса;

результат операции изменяет весь контекст (форма перерисовывается, пересчитываются данные)


\## Встроен в контент

действие локальное и не ломает весь контекст;

пользователь может продолжать взаимодействие с UI;

loader связан визуально с элементом (кнопка, блок настроек);

повторное нажатие по кнопке с loader’ом игнорируется
